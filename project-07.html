Project 07

Portfolio Details

Web Application with Amazon RDS in Private Subnet
(Secure 2-Tier Architecture)

AWS Cloud Platform

September 2024 Isolated Private Database Layer 2-Tier Secure Data Storage Solution

Project Overview

This project demonstrates the deployment of a secure two-tier web application architecture on AWS, where the web application runs on EC2 instances in a public subnet and the database (Amazon RDS) is hosted in a private subnet. The objective was to design a production-ready setup that follows best practices for network isolation, security, and data protection, ensuring that the database is never directly accessible from the internet.

🎯 Objectives

  • Deploy a web application in a public subnet
  • Launch an Amazon RDS database in a private subnet
  • Restrict database access to application servers only
  • Implement secure networking using VPC, subnets, and security groups
  • Follow real-world 2-tier application architecture

🌍 Environment Details

  • ☁️ Cloud Provider: AWS
  • ☁️ Region: ap-south-1 (Mumbai).
  • ☁️ Availability Zones: ap-south-1a, ap-south-1b
  • ☁️ Network: Custom VPC with public and private subnets

🧱 Architecture Components

  • 🏗️ Public Subnet

    • 🔹 EC2 Web Server (Apache / NGINX)
    🔹 Public IP for internet access
    🔹 Internet Gateway attached to VPC

  • 🏗️ Private Subnet

    • 🔹 Amazon RDS (MySQL / PostgreSQL)
    🔹 No public IP
    🔹 Accessible only within VPC

  • 🏗️ Amazon RDS

    • 🔹 Managed relational database service
    🔹 Automated backups
    🔹 High availability (Multi-AZ optional)
    🔹 Secure access using security groups

  • 🏗️ Security Groups

    • 🔹 Web SG: Allows HTTP/HTTPS from internet
    🔹 DB SG: Allows DB port (3306 / 5432) only from Web SG

  • 🏗️ Route Tables

    • 🔹 Public route table → Internet Gateway
    🔹 Private route table → No internet access (or NAT if required)

🔁 Traffic Flow Explanation

    🔹 User accesses the web application via browser
    🔹 Request reaches EC2 instance in public subnet
    🔹 Web application queries Amazon RDS
    🔹 RDS responds to EC2 within private subnet
    🔹 Database is never exposed to the internet

🔐 Security & Best Practices Implemented

  • 🛡️ Database hosted in private subnet
  • 🛡️ No public access to RDS
  • 🛡️ Strict security group rules
  • 🛡️ Network isolation using subnets
  • 🛡️ Principle of least privilege

Through this project, I learned how to design a secure two-tier application architecture using AWS. I gained hands-on experience in deploying web servers in public subnets and hosting databases in private subnets using Amazon RDS. This project helped me understand network isolation, secure database access, and how real-world applications protect sensitive data by restricting direct internet exposure.
