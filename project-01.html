Project Overview

The goal of this project was to deploy my personal portfolio website on the cloud using AWS services. I wanted the site to be globally accessible, secure, and automatically updated whenever I pushed new changes to GitHub.

Tools & Services Used:

  • AWS S3 (for static file hosting)
  • AWS CloudFront (for global content delivery)
  • AWS Route53 (for domain management)
  • AWS Certificate Manager (for SSL/TLS security)
  • GitHub Actions (for CI/CD automation)

Approach:

  • Configured an S3 bucket to host static website files.
  • Integrated CloudFront as a Content Delivery Network (CDN) to reduce latency and improve performance worldwide.
  • AWS Route53 (for domain management)
  • Linked a custom domain using Route53 and enabled SSL/TLS certificates through AWS Certificate Manager for secure HTTPS access.
  • Set up a CI/CD pipeline with GitHub Actions to automatically sync updates from my repository to the S3 bucket.

This project taught me how to combine multiple AWS services to deliver a scalable, secure, and globally available application. It strengthened my understanding of cloud infrastructure and automation workflows, which are essential skills for cloud engineering and DevOps roles.

Amol Raut
Project Lead

Outcome:

  • Successfully deployed a fully functional portfolio site accessible worldwide.
  • Achieved secure HTTPS access with reduced latency through CloudFront.
  • Automated updates using CI/CD, eliminating manual deployment steps.
  • Gained hands‑on experience in cloud deployment, DNS management, and automation pipelines.
