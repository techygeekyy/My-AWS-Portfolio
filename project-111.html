Project 11

Portfolio Details

Production-Grade DevOps & Kubernetes (EKS)

AWS Cloud Platform

January 01, 2026 Cloud Infrastructure | AWS Networking Simulated Enterprise Project

Project Overview

This project focuses on designing and implementing a secure, scalable, and production-ready AWS network architecture using a custom VPC with public and private subnets across multiple Availability Zones. The goal was to understand real-world cloud networking, traffic flow, and security best practices used in enterprise environments.

🎯 Objectives

  • Design a custom AWS VPC with proper IP addressing
  • Isolate resources using public and private subnets
  • Enable secure internet access using Internet Gateway
  • Implement high availability using multiple Availability Zones
  • Apply security best practices using route tables and security groups

🌍 Environment Details

  • ☁️ Cloud Provider: AWS
  • ☁️ Region: ap-south-1 (Mumbai).
  • ☁️ Availability Zones: ap-south-1a, ap-south-1b

🧱 Architecture Components

  • 🏗️ VPC:

    • 🔹Custom VPC with CIDR 10.0.0.0/16

  • 🏗️ Public Subnets:

    • 🔹Used for internet-facing components (ALB / Bastion Host)

  • 🏗️ Private Subnets:

    • 🔹Used for application servers with no direct internet access (Database)

  • 🏗️ Internet Gateway (IGW):

    • 🔹Enables inbound and outbound internet access for public subnets

  • 🏗️ Route Tables:

    • 🔹Public route → Internet Gateway & Private route → None (NAT Gateway)

  • 🏗️Security Groups:

    • 🔹Controlled inbound and outbound traffic following least-privilege principles

🔁 Traffic Flow Explanation

  • Inbound Traffic:

    • 🔹Users → Internet → Internet Gateway → Public Subnet

  • Internal Traffic:

    • 🔹Public Subnet → Private Subnet (Application Servers)

  • Outbound Traffic (Private Subnet):

    • 🔹Private EC2 → None (NAT Gateway → Internet)

    ☑️ Private instances never receive direct inbound traffic from the internet.

🔐 Security & Best Practices Implemented

  • 🛡️ No public IPs assigned to private EC2 instances
  • 🛡️ Internet access restricted
  • 🛡️ Subnet-level isolation across multiple AZs
  • 🛡️ Security Groups used as instance-level firewalls
  • 🛡️ Designed following real production standards

In this project, I learned how to design a secure and highly available AWS VPC by separating resources into public and private subnets and managing traffic using Internet and NAT Gateways. I gained practical experience with AWS networking, routing, and security best practices, and understood how real-world cloud environments are designed to ensure scalability, isolation, and controlled internet access
