Secure Resume Delivery Platform with Time-Limited Access

AWS Cloud Platform

April 2025 Zero-Trust Document Sharing Identity-Based Secure Document Access

🎯 Objectives

This project implements a secure, serverless resume delivery system where HR or recruiters submit a request form to receive access to my resume. The system validates the request, stores request data securely, and generates a time-limited pre-signed URL that expires within 48 hours. The resume is delivered securely via email without exposing public access to the file.

Tools & Services Used:

  • Prevent direct public access to resume
  • Implement time-bound secure file access
  • Automate email-based resume delivery
  • Log recruiter requests for tracking
  • Ensure least-privilege access control
  • Demonstrate real-world secure document sharing architecture

🌍 Environment Details

  • ☁️ Cloud Provider: AWS
  • ☁️ Architecture Type: Fully Serverless
  • ☁️ Access Method: HTTPS REST API
  • ☁️ Expiration Logic: 48-hour time-limited access
  • ☁️ Storage: Private S3 bucket
  • ☁️ Runtime: Python (Boto3 SDK)

🧱 Architecture Components

  • 🏗️ API Gateway

    • 🔹Exposes secure HTTPS endpoint
    🔹Receives resume access requests
    🔹Triggers backend Lambda function

  • 🏗️ AWS Lambda (Python + Boto3)

    • 🔹Validates form input
    🔹Stores recruiter details in DynamoDB
    🔹Generates S3 pre-signed URL (48-hour expiry)
    🔹Sends email notification via SES
    🔹Publishes notification via SNS

  • 🏗️ Amazon DynamoDB

    • 🔹Stores recruiter request metadata
    🔹Tracks access timestamps
    🔹Enables audit logging

  • 🏗️ Amazon S3

    • 🔹Stores resume privately
    🔹No public access enabled
    🔹Accessed only via pre-signed URL

  • 🏗️ Amazon SES

    • 🔹Sends secure resume link to recruiter
    🔹Sends confirmation notifications

  • 🏗️ Amazon SNS

    • 🔹Notifies admin about new resume requests
    🔹Enables operational alerting

  • 🏗️ IAM Roles & Policies

    • 🔹Least privilege access control
    🔹Lambda allowed limited S3 + SES + DynamoDB access

🔁 Traffic Flow Explanation

    1. HR submits resume request form
    2. HTTPS request is sent to API Gateway
    3. API Gateway triggers Lambda function
    4. Lambda validates input and logs request in DynamoDB
    5. Lambda generates a secure pre-signed S3 URL with 48-hour expiration
    6. SES sends the secure download link to HR
    7. SNS notifies administrator of new request
    8. After 48 hours, the link automatically expires

🔐 Security & Best Practices Implemented

  • 🛡️ Resume stored in private S3 bucket
  • 🛡️ No public object access
  • 🛡️ Time-limited pre-signed URL access
  • 🛡️ IAM least-privilege permissions
  • 🛡️ HTTPS-only API endpoint
  • 🛡️ Request logging and audit tracking
  • 🛡️ No server infrastructure exposed

Through this project, I gained hands-on experience in secure document delivery using serverless architecture, implementing time-based access control with pre-signed URLs, integrating multiple AWS services, and designing an event-driven workflow with proper IAM security policies.
